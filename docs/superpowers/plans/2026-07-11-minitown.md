# MiniTown Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a polished, playable browser prototype of MiniTown with connected zoning blocks, automatic exterior roads, staged construction, residents, traffic, inspection, and a day/night cycle.

**Architecture:** React composes a HUD around a transformable world viewport. Pure TypeScript helpers calculate placement and road geometry; a focused simulation hook advances time and town state. CSS renders the approved pixel-3D diorama direction with responsive controls and night lighting.

**Tech Stack:** React 19, TypeScript, Vite, Vitest, Lucide React, CSS

## Global Constraints

- One drag creates one connected block containing one, two, or three plots.
- Roads surround only the shared block exterior and never split internal plots.
- Explore mode never places zones.
- Residential, Shop, and Workspace buildings visibly construct, become occupied, and grow.
- The day/night transition remains readable and uses cool ambient colors with warm occupied lights.
- Core controls and UI text remain code-native.
